1. Executive Summary
Alte University-სთვის უნდა აშენდეს AI chatbot + CRM სისტემა, სადაც chatbot არის პირველადი კომუნიკაციის ფენა, ხოლო CRM არის lead-ების, საუბრის ისტორიის, დავალებების, routing-ის, knowledge-base-ისა და analytics-ის ოპერაციული ბირთვი.
მთავარი პრინციპი: AI არ ცვლის CRM მონაცემებს პირდაპირ. AI აბრუნებს structured analysis-ს, ხოლო Business Rules Engine და Services layer იღებს გადაწყვეტილებას lead-ის შექმნაზე, განახლებაზე, task-ზე ან human handover-ზე.
ამ დოკუმენტში გაერთიანებულია არსებული Master Plan და alte.edu.ge-ს საჯარო სტრუქტურის მიხედვით დამატებული პრაქტიკული მოთხოვნები: პროგრამების knowledge map, admissions routing, international student flow, source governance, consent, analytics და ტესტირების სცენარები.

11. Business Rules Engine
სიტუაცია
CRM action
general_info + no contact
Save conversation only; no lead.
program/admission interest + missing contact
Ask missing fields; save AI summary; no lead until minimum contact exists.
program/admission interest + phone/email
Create/update customer; create/update lead; stage New Lead; task due in 24h.
international_admission
Department International Admissions; priority high if country/city present; capture program and intake.
finance_question
If source confirms answer, reply; otherwise task for Finance/Admissions.
human_request
Handover + task due immediately/4h.
technical_issue
Route IT Support; do not create admissions lead unless contact/admission intent exists.
student_service
Route Student Services; mark existing_student_possible; no admissions lead by default.
event_interest/open day
Create campaign lead if contact exists; campaign_tag = event/open_day.

You are working on a new project: Alte AI CRM Chatbot.
Goal:
Build an AI-powered CRM and website chatbot system for alte.edu.ge. The chatbot must not be standalone. It must be connected to the CRM, and every important message, lead, status, task, handover and AI summary must be stored in the CRM.
Use this stack:
- Backend: Python, FastAPI, SQLAlchemy, Alembic, PostgreSQL, Pydantic
- AI: OpenAI API, structured JSON outputs
- Knowledge Base: PostgreSQL + pgvector-ready structure
- Frontend later: Next.js/React, Tailwind, shadcn/ui
- Deployment later: Google Cloud Run
Important rules:
1. Do not build everything at once.
2. Start with backend foundation only.
3. Keep API routes thin. Put business logic in services and engines.
4. AI must never directly mutate CRM data.
5. All important events must be audit logged.
6. Lead creation must follow business rules.
7. Add duplicate detection by phone and email.
8. Add knowledge source versioning fields.
9. Write clean, modular, testable code.
10. Add tests after every phase.
Phase 0 task:
Create the initial backend project structure under backend/app.
Add FastAPI app, config, database connection, health endpoint, SQLAlchemy base, Alembic setup, .env.example and requirements.txt.
Do not add frontend, WhatsApp, Messenger, Instagram, or AI calls yet.
After Phase 0, explain what files were created and how to run the server.
19. Phase Prompts
Phase 1 CRM Core
Create SQLAlchemy models, Pydantic schemas, services, repositories and API routes for Department, User, Customer, Lead, Conversation, Message, Task, Pipeline, PipelineStage, LeadStageHistory, AuditLog. Add duplicate detection, audit logs and tests.
Phase 2 Website Chat Flow
Create /chat/session/start and /chat/message endpoints. Use mock AI analyzer. Save user message, analyze intent, ask missing fields, create lead/task only by business rules, save AI reply.
Phase 3 AI Integration
Replace mock analyzer with ai_service.py using OpenAI API structured outputs. Add strict source-based system prompt, confidence threshold, ai_interactions persistence and graceful error handling.
Phase 4 Knowledge Base
Implement KnowledgeSource and KnowledgeChunk, upload/search endpoints, active/inactive source control, chunking, source IDs in AI context, pgvector-ready fields.
Phase 5 Frontend CRM
Create Next.js frontend pages: login, dashboard, inbox, leads, lead detail, pipeline, tasks, knowledge, settings. Use clean operational UI and backend API endpoints.
Phase 6 Analytics
Add overview, leads by stage/channel/program, AI handover rate, source coverage, response SLA and conversion metrics.
Phase 7 Channels
Add WhatsApp/Messenger/Instagram/Email connectors only after website chat and CRM core are verified.
