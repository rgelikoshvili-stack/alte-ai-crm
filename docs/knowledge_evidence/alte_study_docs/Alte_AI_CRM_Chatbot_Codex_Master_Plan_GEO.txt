1. პროექტის მთავარი მიზანი
უნდა ავაწყოთ Alte.edu.ge-სთვის AI chatbot + CRM სისტემა, სადაც chatbot მომხმარებელთან პირველადი კომუნიკაციის ფენაა, CRM კი ოპერაციული ბირთვი. ბოტი არ უნდა იყოს ცალკე მდგომი ინსტრუმენტი; ყველა მნიშვნელოვანი ქმედება, სტატუსი, lead და conversation history უნდა ინახებოდეს CRM-ში.
მომხმარებელი წერს Website Chat-ში ან მომავალში WhatsApp/Messenger/Instagram/Email-ში.
AI პასუხობს მხოლოდ ცოდნის ბაზაზე დაყრდნობით.
თუ მომხმარებელი დაინტერესებულია პროგრამით, კონსულტაციით ან ჩარიცხვით, იქმნება lead.

9. Business Rules Engine
Lead creation rules:
- general_info + no contact data => save conversation only
- admission_interest => ask missing contact fields, then create/update lead
- consultation_request => create/update lead + task
- finance_question => route Finance or Admissions depending on context
- human_request => handover + task
- agent_submission => extract student data + create lead + link agent profile
- technical_issue => route IT Support
Duplicate rules:
- If phone exists => update existing customer
- Else if email exists => update existing customer
- Else create new customer
Task rules:
- New admission lead => task due in 24 hours
- Human handover => task due immediately or within 4 hours
- Finance question => task for Finance
- International student => task for International Admissions

13. Codex-ისთვის ძირითადი Master Prompt
ქვემოთ მოცემული ტექსტი შეგიძლია პირდაპირ ჩასვა Codex-ში. პირველად ამით დაიწყე. არ მისცე Codex-ს დავალება “ააწყე ყველაფერი ერთად”; უთხარი, რომ იმუშაოს ფაზებად და ყოველი ფაზის ბოლოს გაუშვას ტესტები.
You are working on a new project: Alte AI CRM Chatbot.
Goal:
Build an AI-powered CRM and website chatbot system for alte.edu.ge. The chatbot must not be a standalone bot. It must be connected to the CRM, and every important message, lead, status, task, handover and AI summary must be stored in the CRM.
Use this stack:
- Backend: Python, FastAPI, SQLAlchemy, Alembic, PostgreSQL, Pydantic
- AI: OpenAI API, structured JSON outputs
- Knowledge Base: PostgreSQL + pgvector-ready structure
- Frontend later: Next.js/React, Tailwind, shadcn/ui
- Deployment later: Google Cloud Run
Important rules:
1. Do not build everything at once.
2. Start with backend foundation only.
3. Keep API routes thin. Put business logic in services and engines.
4. AI must never directly mutate CRM data. AI returns structured analysis; services decide actions.
5. All important events must be audit logged.
6. Lead creation must follow business rules:
- simple informational questions should not create leads
- admission/program/consultation interest should create or update a lead
- human handover should create/update lead and task
7. Add duplicate detection by phone and email.
8. Add knowledge source versioning fields, even if vector search is implemented later.
9. Write clean, modular, testable code.
Phase 0 task:
Create the initial backend project structure under backend/app.
Add FastAPI app, config, database connection, health endpoint, SQLAlchemy base, Alembic setup, .env.example and requirements.txt.
Do not add frontend, WhatsApp, Messenger, Instagram, or AI calls yet.
After Phase 0, explain what files were created and how to run the server.
